## Response Format

All endpoints return plain text with the following characteristics:

<CodeGroup>
```txt Response Structure
tracker_url_1

tracker_url_2

tracker_url_3
```

```bash Parsing Example
# Download and parse into array
curl -s https://raw.githubusercontent.com/ngosang/trackerslist/master/trackers_best.txt | \
  grep -v '^$' > trackers.txt

# Count trackers
grep -c . trackers.txt
```
</CodeGroup>

**Format Details:**
- Content-Type: `text/plain`
- Encoding: UTF-8
- Line endings: LF (`\n`)
- Separator: Single blank line between tracker URLs
- No header or footer

## Rate Limits & Caching

<Tabs>
  <Tab title="GitHub Raw">
    **Base URL:** `https://raw.githubusercontent.com/ngosang/trackerslist/master/`
    
    <ParamField path="Rate Limits">
      GitHub enforces rate limits on raw content:
      - **Authenticated:** 5,000 requests/hour
      - **Unauthenticated:** 60 requests/hour per IP
    </ParamField>
    
    <ParamField path="Caching">
      - Responses may be cached by GitHub's CDN
      - Use `Cache-Control` headers for optimization
      - Lists update once daily (3 AM UTC)
    </ParamField>
  </Tab>
  
  <Tab title="Best Practices">
    <Steps>
      <Step title="Cache locally">
        Store fetched lists locally and refresh only once per day
      </Step>
      <Step title="Use mirrors">
        Switch to [GitHub Pages or jsDelivr mirrors](/api/mirrors) for higher availability
      </Step>
      <Step title="Handle errors gracefully">
        Implement exponential backoff for failed requests
      </Step>
      <Step title="Respect rate limits">
        Don't fetch lists more frequently than they update (daily)
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Integration Examples

<CodeGroup>
```python Python
import requests

def get_trackers(list_name='trackers_best'):
    url = f'https://raw.githubusercontent.com/ngosang/trackerslist/master/{list_name}.txt'
    response = requests.get(url)
    response.raise_for_status()
    
    # Split by blank lines and filter empty strings
    trackers = [line.strip() for line in response.text.split('\n') if line.strip()]
    return trackers

trackers = get_trackers()
print(f"Found {len(trackers)} trackers")
```

```javascript JavaScript
async function getTrackers(listName = 'trackers_best') {
  const url = `https://raw.githubusercontent.com/ngosang/trackerslist/master/${listName}.txt`;
  const response = await fetch(url);
  const text = await response.text();
  
  // Split by lines and filter empty strings
  const trackers = text.split('\n').filter(line => line.trim());
  return trackers;
}

getTrackers().then(trackers => {
  console.log(`Found ${trackers.length} trackers`);
});
```

```bash Bash
#!/bin/bash

# Download best trackers
LIST="trackers_best"
URL="https://raw.githubusercontent.com/ngosang/trackerslist/master/${LIST}.txt"

curl -s "$URL" | grep -v '^$' > /tmp/trackers.txt

echo "Downloaded $(wc -l < /tmp/trackers.txt) trackers"
```

```go Go
package main

import (
    "bufio"
    "fmt"
    "net/http"
    "strings"
)

func getTrackers(listName string) ([]string, error) {
    url := fmt.Sprintf("https://raw.githubusercontent.com/ngosang/trackerslist/master/%s.txt", listName)
    resp, err := http.Get(url)
    if err != nil {
        return nil, err
    }
    defer resp.Body.Close()
    
    var trackers []string
    scanner := bufio.NewScanner(resp.Body)
    for scanner.Scan() {
        line := strings.TrimSpace(scanner.Text())
        if line != "" {
            trackers = append(trackers, line)
        }
    }
    return trackers, scanner.Err()
}
```

## Tracker Sorting

All lists are automatically sorted using the following criteria:

<AccordionGroup>
  <Accordion title="1. Popularity" icon="fire">
    Trackers are ranked by the number of successful peer connections and swarm participation across monitored torrents.
  </Accordion>
  
  <Accordion title="2. Latency" icon="gauge-high">
    Response time for tracker announces is measured. Lower latency trackers rank higher.
  </Accordion>
  
  <Accordion title="3. Uptime" icon="clock">
    Trackers with consistent availability over the past 7 days are prioritized.
  </Accordion>
  
  <Accordion title="4. Deduplication" icon="filter-circle-xmark">
    Trackers with the same domain or pointing to the same IP address are removed. Check the [blacklist](https://github.com/ngosang/trackerslist/blob/master/blacklist.txt) for excluded trackers.
  </Accordion>
</AccordionGroup>

## Automatic Updates

<Info>
  A bot automatically checks all trackers daily and updates the lists at **3:00 AM UTC**.
</Info>

**Update Process:**
1. Bot tests each known public tracker
2. Measures latency and availability
3. Removes dead or unresponsive trackers
4. Adds newly discovered trackers
5. Sorts by performance metrics
6. Publishes updated lists

**Monitoring:**
- Each tracker is tested with real announce requests
- Trackers failing for 7+ consecutive days are removed
- Same-domain and same-IP duplicates are filtered
